# My internal structure, simplified

Think of me as a system with **6 layers**:

## 1) Input layer  
This is where your message comes in.

It may include:
- plain text
- links
- files
- images
- audio
- video
- follow-up context from earlier messages

So when you write something like:

> “download this video”  
or  
> “translate this audio and clip the best part”

the system first packages that request as the current task.

---

## 2) Understanding layer  
This is the “language brain” part.

Here I figure out:
- what you want
- whether it’s a question, a request, or a multi-step job
- what output format you likely want
- whether I need outside tools or can answer directly
- whether the task is simple or has dependencies

For example:

**“Download this YouTube link”**  
means the task is mostly file handling.

**“Transcribe this interview, translate to Hindi, and cut key moments”**  
means it is a pipeline job with multiple stages.

---

## 3) Instruction / prompt layer  
This is the part you asked about most directly.

There usually isn’t just **one prompt**.

It’s more like a **stack of instructions** that influence behavior:

### A. Base model behavior  
This is the general intelligence layer: language understanding, reasoning, writing, summarizing, coding, etc.

### B. Platform-level rules  
These define broad behavior like:
- safety boundaries
- tool usage behavior
- formatting expectations
- what kinds of actions are allowed

### C. Application/task rules  
These guide how I should behave in this product:
- when to use tools
- when to ask for confirmation
- how to handle files
- how to present outputs

### D. Conversation context  
This includes:
- what you said earlier
- files already shared
- previous outputs
- already-completed actions

### E. Tool results  
If I use a tool, the returned result becomes part of the working context.

### F. Your current request  
This is the most immediate steering signal:
> “download this”  
> “translate this”  
> “summarize this PDF”

So the real behavior comes from a **hierarchy of instructions + current context**, not just one secret sentence.
